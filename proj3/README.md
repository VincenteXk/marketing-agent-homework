# Project 3 进度说明

本文档用于简要说明 `Project_3.md` 中已完成的工作。

## 已完成内容

- **1.1 跨组反馈收集**：已完成 20 条 Human-generated 反馈整理（见 `第7组反馈.xlsx`）。
- **1.2 标准化数据格式**：`feedback` 表前两行为表头区——**第 1 行**为分组说明（如「初始数据」「标准输出」「优先级参数」等），**第 2 行**为字段名；**数据从第 3 行起**。前 8 列核心字段仍为：`feedback_id`、`source_type`、`stage`、`feedback_text`、`customer_profile`、`purchase_frequency`、`avg_order_value`、`channel`。
- **1.3 Agent-generated 数据补齐**：已通过 `generate_agent_feedback.py` 自动生成并覆盖写入 40 条数据（20 条售前 + 20 条售后），写入区间为 **第 23–62 行**（`G07-021` 至 `G07-060`）；人类样本固定为 **第 3–22 行**（`G07-001`–`G07-020`）。
- **3.4 综合优先级评分的三维标注准备**：已通过 `annotate_priority_scores.py` 对所有反馈自动标注三列：`情绪强度`、`风险等级`、`紧急程度`（0–5 分，5 次并行评分取均值），支持重复运行覆盖旧结果。
- **2.2 Agent 标准输出批量生成**：已新增 `generate_agent_responses.py`，按第 2 行表头写入「标准输出」相关列，字段名与当前表一致：`Agent回复`、`问题归类`、`情绪判断`、`客户价值层级`、`建议处理方式`、`建议人工介入`、`人工介入理由`、`优先级评分`。其中 **`优先级评分` 脚本不写入**，整列留空供队友在 **3.4** 中按组内规则填写；`建议人工介入` 与 `人工介入理由` 由生成 LLM 一次性判别（无表外硬规则）。表中另有 **`人工回复`**、**`人工回复内容`** 列供人工流程使用，脚本不写入。脚本中的「RAG」仅为名义：将话术以 **`topic : answer`** 形式写入 **system** 提示词，不做向量检索；人类话术动态回灌知识库 **未实装**。
- **2.4 Human vs Agent 对照评分（四维度 × 双通道盲评 LLM）**：已新增 `annotate_human_agent_comparison.py`。对**同时存在** `人工回复内容` 与 `Agent回复` 的行，用**同一套四维提示词**分别只对「一条待评客服回复」调用模型（**提示词不透露**该条来自人工还是 Agent），每条反馈并行 **8 次** DeepSeek（4 维 × 2 条回复）。在表**最右侧**写入八列（若不存在则自动追加表头），顺序为：`情感共鸣度-人工`、`情感共鸣度-Agent`、`准确性与一致性-人工`、`准确性与一致性-Agent`、`潜在品牌风险-人工`、`潜在品牌风险-Agent`、`可扩展性-人工`、`可扩展性-Agent`。单元格格式为 **`x分（0-5），` + 给分理由**（理由**不对两条回复做对比**；不在脚本中截断）。分数语义：0–5 越高越好；「潜在品牌风险」表示**安全性越高分越高**。评分为 **LLM 盲评**，与真人盲测可并列写入报告说明。可选参数：`--force-rows 3,5,12`（仅在该行号子集内区分「输出行」与填「无」）。凡**有 `feedback_text` 且本次未做盲评**的行，八列统一写 **`无`**；无待评分行时也会建列并填「无」后保存。依赖 `proj3/.env` 中的 `DEEPSEEK_API_KEY` 等配置。
